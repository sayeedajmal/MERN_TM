ABSTRACT

The Cloud-Based Task Manager is a comprehensive web application designed to enhance team productivity and streamline task management processes. Leveraging the modern MERN stack (MongoDB, Express.js, React, and Node.js), the system provides a centralized platform for creating, assigning, and tracking tasks in real-time. It features a robust role-based access control system, distinguishing between Administrators, who manage users and oversee global task distribution, and Team Members, who focus on execution and updates. Key functionalities include sub-task management, priority-based leveling, automated activity logging, and secure asset storage via Firebase Cloud Storage. The application addresses the limitations of traditional manual task tracking by providing a responsive, intuitive interface that fosters collaboration and ensures accountability within dispersed teams.

CHAPTER-1

1.1 INTRODUCTION
In today's fast-paced corporate environment, effective task management is the backbone of organizational success. The "Cloud-Based Task Manager" is a full-stack solution built to facilitate seamless task coordination. By utilizing a web-based architecture, it allows team members to stay aligned regardless of their physical location, making it ideal for remote or hybrid work cultures.

1.2 OBJECTIVES
- To provide a centralized repository for all team-related tasks.
- To implement role-based access control to ensure data security and organizational hierarchy.
- To enable real-time tracking of task progress from 'To-Do' to 'Completed'.
- To facilitate detailed communication through task activity logs and comments.

1.3 PROJECT INTERFACE
The interface is designed using React and Tailwind CSS, focusing on a premium, clean aesthetic. It includes a comprehensive analytics dashboard with interactive charts, a managed task list with filtering options, and detailed modals for task creation and editing.

1.4 BACKGROUND
Traditional task tracking often relies on spreadsheets or standalone email threads, which quickly become disorganized. With the rise of the MERN stack, it has become possible to build highly scalable and reactive applications that handle complex data relationships efficiently.

1.5 PROBLEM STATEMENT
Teams often struggle with "who is doing what" and "when is it due." Manual systems lead to missed deadlines, lack of accountability, and poor visibility into the team's total workload. There is a critical need for a tool that automates assignment and provides a single source of truth.

1.6 AIMS AND OBJECTIVES
The primary aim is to build a scalable Task Management System that:
- Minimizes administrative overhead in task delegation.
- Provides visual progress indicators for project managers.
- Secures user data and task assets in the cloud.

3.2 INSTALLATION
Server Setup:
1. Navigate to the /server directory.
2. Run 'npm install' to fetch dependencies.
3. Create a .env file with MONGODB_URI, JWT_SECRET, and PORT.
4. Execute 'npm start' to boot the backend.

Client Setup:
1. Navigate to the /client directory.
2. Run 'npm install'.
3. Configure .env with VITE_APP_BASE_URL and Firebase API keys.
4. Execute 'npm run dev' to launch the frontend.

CHAPTER-4

4.1 PROBLEM DESCRIPTION
The core problem is the fragmentation of task data. When information is scattered across emails and chats, it is impossible to generate accurate reports on team performance. This project centralizes this data into a structured schema.

4.2 USER MANAGEMENT MODULE
This module handles the core "Who" of the application. It includes:
- Authentication: Secure login and signup flows with password encryption.
- Role Management: Distinction between 'Admin' (can create/delete/manage) and 'User' (can update status/comment).
- Profile Customization: Ability for users to update their titles and personal information.

4.3 TASK MANAGEMENT MODULE
This is the functional core of the project:
- Task CRUD: Full Create, Read, Update, and Delete capabilities.
- Delegation: Assigning multiple team members to a single task unit.
- Sub-tasks: Breaking down complex objectives into smaller, trackable items.
- Assets: Attaching visual references or documents to tasks.
- Trashing: A soft-delete system that allows for recovery or permanent removal.

The system must support simultaneous users, handle file uploads up to standard web limits, and provide sub-second response times for dashboard updates. It follows the REST architectural style for API communication.

CHAPTER-6

6.1 PROBLEM DISCUSSION
A significant challenge in real-time applications is ensuring that data remains consistent across different clients. The project uses Redux Toolkit to maintain a single source of truth on the frontend, which syncs with the MongoDB database via Axios.

6.2 IMPLEMENTATION OF PROBLEM
The backend is implemented using Node.js and Express. It uses specialized middleware for Authentication (protectRoute) and Authorization (isAdminRoute). Tasks are stored with a 'team' array containing User ObjectIDs, enabling relational-style queries in a NoSQL environment.

9.1 CODING

--- Server Model (taskModel.js) ---
import mongoose, { Schema } from "mongoose";
const taskSchema = new Schema({
  title: { type: String, required: true },
  date: { type: Date, default: new Date() },
  priority: { type: String, default: "normal", enum: ["high", "medium", "normal", "low"] },
  stage: { type: String, default: "todo", enum: ["todo", "in progress", "completed"] },
  team: [{ type: Schema.Types.ObjectId, ref: "User" }],
  subTasks: [{ title: String, date: Date, tag: String, isCompleted: Boolean }],
  assets: [String],
  isTrashed: { type: Boolean, default: false },
}, { timestamps: true });
const Task = mongoose.model("Task", taskSchema);

--- Server Route (taskRoute.js) ---
import { createTask, getTasks, updateTask } from "../controllers/taskController.js";
import { isAdminRoute, protectRoute } from "../middleware/authMiddleware.js";
const router = express.Router();
router.post("/create", protectRoute, isAdminRoute, createTask);
router.get("/", protectRoute, getTasks);
router.put("/update/:id", protectRoute, isAdminRoute, updateTask);

--- Frontend State Action (Redux Slice example) ---
export const getTasks = createAsyncThunk("tasks/getTasks", async (payload) => {
  const { data } = await axios.get(`${baseUrl}/tasks`, payload);
  return data;
